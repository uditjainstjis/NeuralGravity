Why It Slows Down

Sequential verification

Batch verification produced numerical drift on MLX/Metal, so the final correct version verifies drafted tokens one by one to match the baseline execution path.

Draft re-prefill

The drafter rebuilds context each iteration, which turns accepted-prefix growth into repeated work.

Python dispatch

The correctness-first loop adds repeated Python-to-Metal round-trips instead of collapsing work into one efficient target pass.

Useful Takeaways

Validate correctness before timing

Speculative decoding can appear stable on several prompts and still diverge later.

Do not inherit GPU assumptions

On this stack, larger speculative depth made throughput strictly worse.

Negative results are still useful

This repo now provides a reproducible baseline for future MLX and Apple Silicon inference work.
