## 1. Project Objective
The primary goal was to achieve "Neural Gravity"—a framework for running and fine-tuning 3B-parameter models (e.g., Qwen2.5-3B) within the strict 5.4GB Metal memory limit of an 8GB MacBook. The focus was on ensuring stable reasoning capabilities on ultra-low bit (4-bit/2-bit) quantized substrates.

## 2. Key Innovation: Hybrid Low-Rank Adaptation (HLRA)
We developed **HLRA**, a dual-path adapter designed to compensate for the catastrophic weight destruction caused by aggressive quantization (PTQ).

- **Path 1 (DoRA)**: Handles standard weight-decomposed directional updates.
- **Path 2 (EoRA)**: A residual path explicitly initialized via **Singular Value Decomposition (SVD)** of the *true* quantization error matrix $E = W_{orig} - Q(W)$.
- **Status**: **Successful Validation**. Over 200 training steps, HLRA demonstrated significantly deeper and more stable loss convergence than standard LoRA or DoRA on 2-bit models.
- **Artifacts**: `neural_gravity/hybrid_adapter.py`, `reports/hlra_loss_curve.png`.

## 3. The Speculative Cascade Experiment (Attempted & Pivoted)
We initially attempted to achieve 1.5x+ TPS speedups using **Speculative Decoding** (0.5B Drafter + 3B Target).

- **Approach**: 5-step draft generation with parallel target verification and KV-cache rollback.
- **Attempt 1 (Python Loop)**: Successful implementation, but **Failed performance** (-58% speedup). TPS dropped to ~16.6 (vs 40.0 baseline) due to MLX JIT overhead and constant interrupts between Python and Metal.
- **Attempt 2 (Custom Metal Kernel)**: Wrote a fused Q4 attention kernel in C++/Metal using `mx.fast.metal_kernel`. 
- **Result**: **Failed performance** (~13.5 TPS). This confirmed the bottleneck was system-level coordination/dispatch, not raw GPU attention math.
- **Outcome**: The project **Hard-Pivoted** away from speculation. We rewrote the ICLR 2026 paper to focus exclusively on the HLRA adapter logic.
- **Artifacts**: `fast_metal_cascade.py`, `benchmark_uma_cascade.py`.

## 4. Current Work: Test-time A* Search (TTA*)
To recover reasoning capacity in the 3B model without retraining, we are implementing a **Test-time A* Search** decoding wrapper.

- **Status**: **Work-In-Progress (Debugging)**.
- **Mechanism**: $f(n) = g(n) + h(n)$, where $g$ is cumulative log-prob and $h$ is a self-reflection heuristic score generated by the model rating its own thoughts.
- **Current Blocker**: Finalizing 1D/2D tensor shape compatibility within the `generate_step` loop for A* expansion nodes.
- **File**: `reasoning_search.py`.
